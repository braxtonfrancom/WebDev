Software Development Plan

Phase 0: Requirements Specification

My goal is to build a webpage that will be able to tell someone the dollar value of a weight in gold. I also would like to have my website be able to convert between different units of conversion including lbs, oz, troy oz, grams, kilagrams, and U.S. Tons.

I haven't worked much with Javascript but I am actually really excited to become more familiar with it because it is so common. I think the biggest challenge that I forsee is figuring out what to do in HTML and what to do in Javascript. I feel like I could do most things in either but I am not sure what is the ideal way to approach individual tasks.

Phase 1: System Analysis

I will be taking in a number input and the user will select a unit of conversion from a dropdown menu. I will need to limit the input to numbers, otherwise I will run into errors trying to perform operations on said input. I will then need to appropriately convert the the units into whatever the user requested.

My output will be the dollar amount of the weight entered in gold based on the current gold value.

There really won't be any algorithms or formulae used in this program. The only 'algorithms' of sorts will be the simple conversion factors used in this program.

Phase 2: Design

  • gold app:
    • This app will be done mostly in a javascript script. I will make the base layout using DOM manipulation. There will br two fetch() calls. One before the button is clicked to get the price of gold from an API. The other will be called once the button is clicked. This one will call my unit conversion API to convert the selected units to troy ounces. When the '=' button is clicked, it will call the followiong function:
    • convert()--> This function will make a fetch() to my unit conversion API. It will get the weight in troy ounces which we can multiply with the current price of gold. We will then display this calculated value in a new div that we add to the DOM.
  • unitconv app:
    • This app will have very minimalist HTML body. The work for this app will be done in views in a function called convert().
    • convert(request)--> This function will take a request in. After ensuring that it contains all the necessary pieces I will calculate a value based on the values contained within the request. It will be something like float(value) / from * to. This will give us the value of the converted number. I wll then return the value of the converted number and the units we are converting to in a JsonResponse.

Phase 3: Implementation

I implemented all of the above code and more to create a functional webpage that can perform the above described operations.

I learned once again that Javascript is a very powerful tool. I also learned that it's syntax can be decieving at times. It can be quite tricky at times figuring those things out but this project helped me to develop a lot more confidence in my ability to use Javascript as well as become more comfortable working with URL's.

Phase 4: Testing and Debugging

Test Case: I tested putting 4 U.S Tons into the gold app by typing 4 into the input box and selecting U.S Tons from the dropdown menu.

Result: A new div appeared at the very top beneath the main div. It displayed 'At Fri Nov 11 2022 19:26:14 GMT-0700 (Mountain Standard Time), 4 tons of gold is worth $205,887,585.79'. This was the expected output.

Test Case: I typed 'http://localhost:8000/unitconv/convert?from=g&to=lb&value=76' into my browser.

Result: The webpage returned the following JSON object: {"units": "lb", "value": 0.16755039706780697}. This is the expected output.

Test Case: I pressed the '=' button on the gold app without entering a value into the input box.

Result: A new div appeared at the very top beneath the main div. It displayed 'Fri Nov 11 2022 19:35:59 GMT-0700 (Mountain Standard Time) You didn't enter anything man!'
