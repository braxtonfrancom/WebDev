Phase 0: Requirements Specification

My goal is to build a fully functional, dynamic Django website. The website will have a homepage, an archive page that will show all of the blog posts, a entry page that will show a specific blog post and will have a form to add a comment to the page, and then I will have my about.html, techtips+css.html, and techtips-css.html pages as well.

To be honest, I am pretty unfamiliar with everything. Django is very foreign to me and it is going to be a big learning curve. Obviously, I have already written the HTML and css for some of my webpage, but I have a lot of learning to do in order to be able to understand how to work those into the Django framework.

Phase 1: System Analysis

Some of the data I will be using in this program will be taken from our last assignment. I am going to be using HTML files styled with CSS. I will have a form in one of the HTML files that will POST data to the server. I will want to put this data into our database as well as display it dynamically on our webpage.

My output will be a fully functioning, dynamic Django website.

Phase 2: Design

  • The first step for me is to bring in my about.html, plan.html, techtips+css.html, and techtips-css.html files and get them converted from static pages to dynamic django pages.
  • My next step is to make a blogHome.html, archive.html, and entry.html files. In addition to this, I need to set up all of my models.py, urls.py, and views.py files. I will break down the functions and classes that are going to be in these files in the following paragraphs.
  • models.py
    • class Blog(models.Model)

      In this class I will need to make several class attributes that will make up a Blog object. These attributes will be a title_text(models.CharField()), a author(models.CharField()), a content(models.TextField()) and a posted(models.DateTimeField())

      I will also need a method __str__(self) to return a string representation of the Blog object.

    • class Comments(models.Model)

      In this class I will need to make several class attributes that will make up a Comments object. These attributes will be a blog(models.ForeignKey()), a commenter(models.CharField()), a email(models.EmailField()), a content(models.TextField()) and a posted(models.DateTimeField())

      I will also need a method __str__(self) to return a string representation of the Comments object.

  • urls.py
    • In urls.py I actually won't need to make any functions or classes. I will just need to add a handful of paths() to the urlpatterns[] list so that all of my webpages are accessible.
