Software Development Plan

Phase 0: Requirements Specification

My goal is to build a fully functional weather app using Vue.js as a framework.

I am excited to work with Vue.js. I have heard a lot of good things about it. I think the biggest challenge for me in this assignment will be learning how Vue.js works. Once I figure that out, I don't think the implementation of the rest of the assinment will be that difficult.

Phase 1: System Analysis

I will be using multiple API's to get data relating to the weather and forecast. This data will be returned as a json object.

My output will be a webpage that displays all of the information I extract from the json objects.

There really won't be any algorithms or formulae used in this program. The program will mostly be fetch() calls and then accessing the data in the json objects that those fetch()'s return.

Phase 2: Design

  • I will have a script.js, index.html, and style.css page.
  • script.js
    • I am going to leave this file blank. For this project, I found it more convienient to have my script inline in index.html
    index.html
    • In index.html I am going to have a basic html layout with dynamic Vue data inserted throughout. The individual divs in this layout will be conditionally rendered using Vue directives.

    • In my Vue.createApp() I will have the following four sections:
    • 1. data() - In here I will define any data that will need to be passed into the html body as a dynamic element. Some of this data will be json objects that will come from API calls. I will also have a "error" data element that I will set to undefined. If an error develops while the API fetch() calls, the error message will be printed to the screen because of conditionals I will have in my html layout that check to see if error === undefined.
    • 2. methods() - I think I will only need one method in here. It it will be a method that takes in an event(such as a click). It will then look at the current state of the div that was just clicked and determine the color that it needs to be changed to.
    • 3. computed() - Here, I will have three methods. The three methods will be nearly identical. They will be unlikely(), likely(), and neutral(). Their job will be to keep the count of the repective states of all the forty divs.
    • 4. created() - Everything in here will be automatically called when the app is created. In here I will have three API calls. The first will be to get the current location of the person using the website. The second will use that location to get the current weather for that location, and the third will be to get the forecast for the next five days for that location. I will promise chain these API calls so that I can catch errors simply and efficiently.
    style.css
    • My stylesheet will have basic css selectors that will style my weather app.

Phase 3: Implementation

I implemented all of the above code to create a functional weather single-page web application.

I learned that Vue.js is a very functional framework. Once you learn how it works, it's implementation is actually pretty simple. It can be very powerful and can make dynamic webpages simple.
