Phase 0: Requirements Specification

My goal is to build a fully functional HTTP server. It will be able to interpret HTTP requests and form valid responses. It will also be able to use HTTP status codes to control the browser and redirect requests to different resources as well as be able to send custom response headers. In addition to these requirements, I would like my server to be able to serve HTML, CSS, and image files and properly display them. Lastly, I would like my server to be able to serve dynamically-generated pages, and gracefully handle client caused errors.

To be honest, I am pretty unfamiliar with everything. Obviously, I have already written my HTML and CSS webpages so I am quite familiar with that aspect of the project, but I have a lot of learning to do in order to be able to understand how I am going to serve these webpages to my server.

Phase 1: System Analysis

The data in this program will be taken from our last assignment. We are going to be using HTML, CSS, and JPEG files. We will 'serve' these files onto the server which will have to be able to properly handle all of the different formats.

My output will be a fully functioning HTTP server. It will properly load all of the pages and navigate around the site effectively. It will be able to smoothly handle both correct and incorrect searches in the 'search bar'.

There won't be any algorithms or formulae used in this program.

Phase 2: Design

  • do_GET(self):

    This function will inspect self.path and then respond in the correct manner.


    To do this, I will have if/elif/else statements that will inspect self.path and respond accordingly.


    There will be no parameters passed into the function besides self


    In the case of good input: The function will go to the correct if/elif statement and will send the appropriate headers.

    In the case of bad input: The function will default to the else statmetn and will return a 404 Not Found error.

  • openFile(self, string(string)):

    This function will take a string file name into it. It will then read that file and then close the file. At this point I will send a Content-Length header by taking the length of the opened file. Then I will need to end headers and then write this opened file to the server.


    To do this I will need to use open(), read(), close(), and then use self.send_header, self.end_headers(), and self.wfile.write()


    The only parameter passed into the function will be a string. This string is a filename that I will use in open()


    In the case of good input: The function will take the string which is a file name and open it. It will then send extract some information(Content-length) and send it as a header. Then the opened file will be served.

    In the case of bad input: The bad input will never reach this point because if it is an invalid file name, openFile() will never be called. It is only called when there is a valid filename given.

  • sendHeaders(self, contentType(string), statusCode(string)):

    This function will send the appropriate headers when a html request is given. It will determine which headers need to be sent based on the statusCode.


    I will have several if/elif statements that send different status responses

    Then I will have several self.send_header's that all send unique headers


    The contentType parameter will be used when sending the Content-Type header. This will determine if the file is a jpeg, html, css, etc.


    In the case of good input: The function will send the appropriate headers.

    In the case of bad input: The function will hit the else statement which will send a 404 code. This will handle all invalid input.
